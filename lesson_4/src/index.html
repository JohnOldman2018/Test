Нумерованный список (ordered list)

  <style>
    ol {list-style-type: upper-roman;}
    ol ol {list-style-type: upper-alpha;}
    ol ol ol {list-style-type: decimal;}
  </style>
  1. Introduction
  2. Main body
    1. Ice-breaker cartoons
    2. Examples
      1. Personal experiences
      2. What friends say
  3. Appendages

Наиболее популярными значениями для свойства list-style-type для упорядоченного списка являются: decimal, upper-alpha, lower-alpha, upper-roman, lower-roman.

Для элементов <ol></ol> иногда используют атрибуты reversed и start

Организационные элементы

  • section
  • article
  • aside
  • nav
  • header
  • footer

Обычно нет необходимости использовать организационные элементы для небольших веб-страниц, но для многостраничных веб-сайтов их следует использовать согласованно. Организационные элементы являются блочными элементами, то есть расширяются на всю ширину содержащего их контейнера. Это можно увидеть по зеленому цвету фона заголовков на рисунке ниже, который показывает, что содержимое контейнеров <section></section> растягивается на всю ширину содержащего их контейнера <article></article>

Контейнер <section></section> тематически группирует вместе раздел веб-страницы. То есть содержимое контейнера связано общей темой.
Обычно, контейнер содержит в себе заголовок и один или несколько параграфов или списков, объединенных одной тематикой, указанной в заголовке.

Контейнер <article></article> служит для группирования вместе одного или нескольких контейнеров <section></section> таким образом, что группа таких разделов формирует отдельную независимую часть веб-страницы.

Следующий код демонстрирует все организационные элементы:


    <!DOCTYPE html>
    <html lang="en">
    <head>
    <meta charset="utf-8">
    <meta name= "author" content="John Dean">
    <title>Mangie's List</title>
    <style>
      body {background-color: lightyellow;}
      header {background-color: powderblue;}
      section > h2 {background-color: palegreen;}
      aside > h3 {margin: 0;}
      aside {
        border: thin dashed red;
        color: red;
        background-color: white;
        float: right;
        width: 200px;
        margin: 5px;
        padding: 10px;
      }
      address {display: inline;}
    </style>
    </head>

  <body>
  <header>
    <h1>Mangie's List</h1>
    <h2><q>Simply the best reviews anywhere!</q></h2>
  </header>
  <nav>
    <a href="#dining">Dining</a> | <a href="#clothing">Clothing</a>
  </nav>
  <article>
    <section>
      <aside>
        <h3>Casey's Special</h3>
        Half-price hot dogs when rotisseried more than 24 hours!
      </aside>
      <h2 id="dining">Dining</h2>
      <ol>
        <li>China Star</li>
        <li>Casey's General Store</li>
      </ol>
    </section>
    <section>
      <h2 id="clothing">Clothing</h2>
      <ol>
        <li>Eddie Bauer</li>
        <li>Kansas Speedway Boutique</li>
        <li>Casey's General Store</li>
      </ol>
    </section>
  </article>
  <footer>
    Questions? Email <address>mangie@gmail.com</address>.
  </footer>
  </body>
  </html>

Элемент <aside></aside> группирует вместе информацию, которая имеет отношение к следующему за ним элементу, но не является частью основного содержания раздела. Обычно данный элемент располагается справа или слева от основного текста. В вышеприведенном примере этот элемент находится справа от соответствующего раздела и выделен пунктирной красной границей. Несмотря на то, что данный элемент является блочным, браузер показывает его в компактном виде. Это достигается благодаря следующему участку кода CSS: 

      aside {
        border: thin dashed red;
        color: red;
        background-color: white;
        float: right;
        width: 200px;
        margin: 5px;
        padding: 10px;
      }
Строчка float: right; помещает элемент <aside></aside> справа от остальных элементов контейнера <section></section>, а строчка width: 200px; уменьшает ширину данного элемента до 200 пикселей.

Элемент <nav></nav> использует элементы <a></a> для навигации по веб-странице

Элемент <footer></footer> часто содержит в себе такие данные, как авторское право, информация об авторе, связанные ссылки и т.д. В данном элементе часто располагают элемент <address></address>, который предоставляет контактную информацию для всей веб-страницы. Если элемент элемент <address></address> размещают внутри элемента <article></article>, то контактная информация дается только для статьи <article></article>. Сам по себе элемент <address></address> является блочным элементом и чтобы его поместить в ряд с другими элементами, необходимо изменить его поведение. 

      address {display: inline;}

Селекторы, которые обозначают любого потомка данного элемента, просто отделяются пробелом от селектора предка. Селекторы, которые обозначают ребенка данного элемента, отделяются от селектора родительского элемента знаком > (combinator) отделенного от окружающих селекторов пробелами:
section > h2 {background-color: palegreen;}

Если элемент <a></a> ссылается на другую страницу, а не на место в текущей странице, то в первом теге этого элемента необходимо поместить атрибут target вместе с его значением "_self" или "_blank". Значение "_self" делает переход по данной ссылке (по умолчанию) в том же окне браузера, а значение "_blank" приводит к тому, что открывается новое окно браузера.
