Селекторы псевдо-классов

Использование селекторов псевдо-классов позволяет избежать вставки пары class="value" в начальные теги нужных элементов по всей веб-странице, если нам вдруг понадобилось выделить определенную группу элементов в коде CSS. Например, строчка tr:first-of-type {background-color: palegreen;} выделяет фоновым цветом все первые элементы в какой либо группе, в примере это элемент <tr></tr> первого ряда в таблице. Иначе говоря, псевдо-классы позволяют отказаться от использования атрибута class. Псевдо-классы обеспечивают выбор среди дочерних элементов.

Псевдо-класс :nth-of-type() является наиболее гибким. В скобках может быть целое число, указывающее на элемент. Может стоять слово even или odd, указывающее на четные или нечетные элементы. Чтобы облегчить чтение таблиц программами для слепых пользователей и поисковых машин, ячейки, которые соответствуют одновременно разным ячейкам над ними, снабжаются атрибутом headers, которому присваивают значения, соответствующие идентификаторам id, заранее присвоенным вышележащим ячейкам. Но правильным способом использования табличного представления данных для их визуального выравнивания и отображения на экране является применение одного из двух способов с использованием CSS:
в первом способе применяется свойство display к табличным значениям и тогда границы ячеек в таблице будут увеличиваться и сжиматься, как это делается для элемента HTML table
во втором способе используются свойства позиционирования CSS и тогда границы остаются фиксированными.

1. Таблица значений свойства display

До сих пор мы использовали свойство display в паре display: inline для визуализации элемента address (который в норме является блочным элементом). Но это свойство можно также использовать для эмуляции различных частей таблицы. 
  <style>
    .table {display: table;}
    ...
  </style>
  <body>
  <div class="table">
    ...
  </div>
  </body>
В данном коде для элемента div используется селектор класса table. Можно использовать любое другое название селектора, чтобы не путать его со значением свойства в паре: display: table;. Данное значение заставляет любой выбранный элемент вести себя, как таблица.

2. Абсолютное позиционирование с помощью свойств позиционирования CSS

При таком позиционировании размеры ячеек таблицы будут фиксированными. Это необходимо, например, если стоит задача выдерживать одну и ту же ширину газетных колонок в статье.

У браузеров по умолчанию используется схема компоновки normal flow (нормальный поток), когда блочные элементы располагаются сверху вниз отдельно каждый в своем ряду, а фразовые элементы располагаются справа налево, занимая свободное пространство.
Чтобы вывести элемент из нормального потока, можно использовать свойства позиционирования CSS, показанные в таблице ниже.

Свойства позиционирования CSS
Свойства позиционирования CSSОписание
position: absoluteЗаставляет браузер располагать элемент относительно содержащего его блока (блока контейнера). В норме блок контейнера является элементом <html></html>.
left: значение (px или %)Задает расстояние между левым краем блока контейнера и левым краем элемента (где левый край означает слева от любого padding, border или margin).
top: значение (px или %)Задает расстояние между верхним краем блока контейнера и верхним краем элемента (где верхний край означает выше любого padding, border или margin).
width: значение (px или %)Задает ширину невидимого блока, который ограничивает содержимое элемента. Ширина не включает в себя padding, border или margin).
height: значение (px или %)Задает высоту невидимого блока, который ограничивает текст элемента. Высота не включает в себя padding, border или margin).

Свойства top и left определяют, как верхний левый угол целевого элемента позиционируется относительно блока контейнера элемента. В норме блок контейнера является элементом <html></html>.

Если для элемента с абсолютным позиционированием вы не укажете ширину элемента (свойство width), тогда ширина элемента будет определена естественной шириной содержимого элемента. Если содержимое является изображением, то это приемлемо, поскольку в норме вы хотите, чтобы изображение отображалось в своих первоначальных размерах.
С другой стороны, если содержимое является значительным фрагментом текста, вы захотите указать ширину элемента. Почему? Если вы не укажете ширину, то браузер попытается расширить элемент, чтобы показать весь текст в одной строке без разрывов.
То же самое применимо к свойству height.

Следующий код создает схему компоновки текстовых элементов, похожую на газетные столбцы: 

<style>
  .title {
    position: absolute;
    left: 10px; top: 4940px;
    width: 620px; height: 30px;
    text-align: center;
    font-size: 20pt;
  }
  article {
    position: absolute;
    top: 5000px;
    width: 300px; height: 180px;
    text-align: justify;
    font-size: 12pt;
  }
  article.left {left: l0px;}
  article.right {left: 330px;}
</style>

<article class="left">

  at the Thompson Center, October 22 from 10 pm to midnight.
</article>
<article class="right">

</article>

Относительное позиционирование

In Divine Comedy, Dante describes the height of ecstasy and the depths of despair.


Элементы смещаются относительно содержащего их контейнера. В данной фразе две её части сдвинуты по вертикали, а вторая еще и по горизонтали с помощью следующего кода CSS: 

<style>
  .up {
    position: relative;
    top: -1em;
  }
  .down {
    position: relative;
    top: 2em;
    left: 2em;
  }
 </style>


Обе сдвигаемые части фразы помещены в элементы <span></span> которым присвоены классы up и down. 

Относительное позиционирование используется редко.
